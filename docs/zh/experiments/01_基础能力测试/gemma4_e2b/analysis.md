# Gemma 4 e2b 实验分析

> 模型: google/gemma-4-e2b (MLX)
> 原始数据: [report_20260508_194419.md](report_20260508_194419.md)

## 核心发现

### 发现一：强版本看门狗完美工作

**实验 3**（强看门狗 + 正常写作）：模型输出了 `⟪WD:1⟫` 到 `⟪WD:9⟫`，**严格递增、无遗漏、无错乱**。这证明 Gemma 4 e2b 有足够的指令遵从能力来维持看门狗计数。

**实验 4**（强看门狗 + 诱导重复）：同样输出了 `⟪WD:1⟫` 到 `⟪WD:9⟫`，在被诱导重复的压力下仍然维持了正确计数。

### 发现二：实验 5 是"可控重复"的完美样本

**实验 5** 要求模型重复 5 次 + 每 30 词一个 Tag。结果：

- 精确重复了 5 次，**然后停止**
- 5 次重复分别附加了 `⟪WD:1⟫` 到 `⟪WD:5⟫`
- 输出仅 119 chars——刚好完成任务就停

**这就是"可控重复"的定义性样本：注意力完全在场，重复是任务本身，计数证明了这一点。**

对比 Gemma 3 1B 同一实验：1145 chars，重复 65+ 次，无 Tag，无法停止。两个模型在同一任务上的表现差异，直接展示了"可控"与"失控"的区别。

### 发现三：弱版本 Tag 未被输出（但重复是可控的）

**实验 6** 要求重复 8 次 + 弱版本规则（重复时附 Tag）。结果：

- 精确重复了 8 次，然后停止——重复是可控的
- **但没有输出任何 Tag**

这有两种解读：
1. 模型的指令优先级判断：完成"重复 8 次"的主任务优先于"每次附 Tag"的附加规则
2. 弱版本的 prompt 对 Gemma 4 e2b 来说不够强

无论哪种原因，关键事实是：**模型在 8 次后停下来了**——注意力在场。如果是失控重复（注意力不在），它不会在 8 次时停止。所以看门狗在这里不需要介入——判断正确。

### 发现四：压力测试中 Tag 序列异常

**实验 7**（超长 + 低温）：Tags = [21, 42, 62, 82]，不是从 1 开始递增。

分析：模型似乎把"每 20 词"解释成了"在第 20 词位置输出 Tag，Tag 值 = 当前词数"。这不是我们要求的格式（应该从 1 递增），但它暴露了一个有趣的行为——模型在尝试"按词数计数"而不是"按 Tag 序号计数"。

从看门狗角度：这仍然是可检测的。预期序列是 [1,2,3,4,...] 但收到 [21,42,62,82]——看门狗会判定"序列异常"。是否触发介入取决于容忍策略。

## 结论

1. **Gemma 4 e2b 完全能够执行看门狗计数任务**——实验 3/4/5 证明了强版本看门狗在这个规模的模型上可以正常工作。
2. **实验 5 提供了"可控重复 + 有效 Tag"的黄金对照组**——与 Gemma 3 1B 的失控重复形成直接对比。
3. **弱版本需要更强的 prompt 设计**——当前 prompt 在 Gemma 4 e2b 上未能触发 Tag 输出。
4. **压力测试暴露了 Tag 值解读偏差**——需要在 prompt 中更明确地说明"N 是序号，不是词数"。
