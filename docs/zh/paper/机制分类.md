# 机制分类与层级

本文档是主论文的框架支撑。它不直接展开"为什么 AgentPreproxy 必要"——那是主论文的事；本文档只把"机制"这个词在本系列论述里的精确定位讲清楚：

1. 它是相对一个分析层级而言的（避免落入"传感器内部还有传感器"那种无穷回归）；
2. 在锁定层级下，它本身又分两个意义；
3. 一条完整机制的内部结构可以分解为四层；
4. 由此，机制层下面分两类——信号机制 vs 反射机制；
5. 这些机制共享一个一般化原则：宏观上的注意力再分配。

主论文里出现的 (A)/(B) 命名、"反射机制"、"自指闭锁"等措辞，含义都来自这里。

## 一、"机制"是相对一个分析层级而言的

"机制"这个词在使用之前必须先把边界划清楚——否则会有一种潜在的误读：每个机制内部都还有更下一层的机制，那是不是要追到原子级别才算真机制？

这个问题真实存在，但解法是干净的。

任何"机制"内部都有自己的子机制。疼痛反射弧是机制，但反射弧本身是神经元放电、离子通道、蛋白质构象——再往下还是机制；继续追下去要追到原子甚至更下，没有自然停止点。这不是论证漏洞，是任何分层模型必然如此。

唯一可操作的处理是：**"机制"不是一个绝对范畴，而是相对一个分析层级而存在的范畴。** 在任何分析里，先选定一个**主体 (agent)**，然后：

> **机制 = 主体之下、主体不能感知和修改的那一层。**

层不一样，机制就不一样：

- "人 vs 自己的身体"这一层，疼痛反射弧是机制（人关不掉它）。
- "细胞生理学"那一层，反射弧不再是机制，它成了研究对象，机制下移到离子通道。
- "分子生物学"那一层，离子通道也下移，机制变成蛋白质折叠。

本系列论述锁定的层级是固定的：

> **主体 = 模型本身**（行为接口是 prompt 与 token 输出）。
>
> **机制 = 模型不能感知和修改的一切**（proxy、Core、底下的硬件……都在这一层）。

硬件之下还有更微观的物理过程，但跟本文论证无关——它跟模型的"自我感知"和"提示词约束"之间不构成直接接口。所以本文不需要回答"什么是终极意义上的机制"，只需要回答"对模型来说，什么在它的接口之下"。这是一个有限、可操作的问题。

## 二、在锁定的层级下，"机制"还分两个意义

层级锁定之后还有第二件事要说清楚：在"机制层"内部，"机制"这个词其实承担了**两种不同的结构角色**。混着用会造成误解，必须分开。

**(意义一) 信号意义**——提供主体本来没有的信号源。例：光线传感器。没有它，主体根本不知道有"光线"这件事。它的作用是**让主体能知道**。

**(意义二) 强制意义**——绕过主体认知、自动触发响应的耦合。例：生物疼痛。它不仅提供信号，还内置一条反射弧——信号传到脊髓直接驱动肌肉收缩，**在认知开始处理之前就已经发生了**。它的作用是**让系统必须做**。

两个意义都成立，但刻画的是不同的结构属性：

- 意义一定位于"信号通路是否存在"——没有它，主体根本看不见这件事。
- 意义二定位于"响应是否绕过认知层强制发生"——没有它，主体的认知层有可能漏接、忽略、或来不及反应。

注意：**两者不互斥，可以叠加**。光线传感器只承担意义一；生物疼痛两者都有（既是新信号源，又内置强制反射）。一条**完整且强力**的机制通常**两层都包含**——先要能知道（意义一），再才能强制做（意义二）。

为什么要拎出这个区分：因为光是有意义一是不够的。在模型注意力已经塌掉的故障态下，再灌一条信号进去也没人接——必须有意义二（强制耦合）才能真正闭环。这件事在 [《主论文》](主论文.md) §3 失控重复的死锁论证里会被推到极端，那也是 §4 把机制分成两类之所以必要的原因。

## 三、一条完整机制的内部结构：四层

前两节分别锁定了"机制"的分析层级和它在这个层级下承担的两种意义。在把机制分成 (A)/(B) 两类之前，还要把**一条机制的内部结构**展开看——否则后面讨论"什么决定了一条机制属于 (A) 还是 (B)"时，容易把不同层次的东西搅在一起。

一条完整的机制可以分解成四个层次：

| 层次 | 名称 | 作用 | 例（"防止机器人撞墙"） |
| --- | --- | --- | --- |
| 第一层 | **不变量** (invariant) | 要保证的目标 | "不能撞上东西" |
| 第二层 | **指标** (indicator) | 量化这个目标 | "距离 < 5cm 算危险" |
| 第三层 | **传感器** (sensor) | 用什么手段获取这个量 | 光线 / 声波 / 激光雷达 |
| 第四层 | **强制耦合** (coupling) | 测出来之后自动触发什么 | 低于阈值 → 强制刹车 |

每一层关心的问题完全不同：

- 第一层是**目标选择**——保什么。
- 第二层是**量化标准**——怎么衡量。
- 第三层是**工程实现**——用什么测。同一个指标可以由不同传感器实现（光线测距 / 声波测距），这只是工程裁量，不是结构差异。
- 第四层是**质变**——有没有强制响应。前三层加起来只是"能知道"；第四层加上去才变成"必须做"。

这个分层跟 §2 的两个意义是对应的：

- **意义一（信号意义）= 前三层**：有了不变量、指标、传感器，主体就"能知道"。
- **意义二（强制意义）= 第四层**：有了强制耦合，系统就"必须做"。

有了这个四层分解，下一节的 (A)/(B) 分类就变成一句话的事。

### 附注：换传感器不等于换类别

四层结构还能解释一个容易混淆的问题：**同一个传感器到底属于 (A) 还是 (B)？**

答案是：**传感器本身既不属于 (A) 也不属于 (B)。** (A)/(B) 分的不是传感器，是**整条控制环路包含哪几层**。

- 光线传感器 + 主控制器读了再决定怎么做 → 环路只到第三层，属于 (A)。
- 光线传感器 + 低于阈值就强制刹车（绕过主控制器）→ 环路到了第四层，属于 (B)。

**同一个传感器能进 (A) 也能进 (B)——决定它属于哪一类的不是它自己，是它接下来跟什么耦合在一起。**

同理，光线测距和声波测距之间的替换发生在第三层内部——是工程选择，不是分类差异。

同一个不变量也可以由多条平行的传感器链共同支撑（传感器融合）。这不会让它变成"多个机制"——不变量只有一个，强制耦合也只有一个，只是输入端由多个传感器联合提供。

## 四、机制层下面其实分两类：信号机制与反射机制

有了前面的铺垫——层级锁定（§1）、两个意义（§2）、四层结构（§3）——机制层的分类就清楚了。核心就是一句话：**(A) 只包含前三层；(B) 包含全部四层。** 展开来说：

**(A) 信号机制**——只包含前三层（不变量 + 指标 + 传感器），不包含第四层。给模型一条新的信号通路，让它"能知道"。模型仍然要靠自己的注意力去用这条信号。光线传感器属于这一类；将外部资料注入上下文（RAG 一类的做法）也属于这一类。

**(B) 反射机制**——包含全部四层。在前三层的信号通路之上，还有第四层——**不经模型认知就触发**的强制耦合。生物疼痛属于这一类；失控重复处理（探针 + 截断 + 注入新信息）也属于这一类——探针提供前三层，截断和注入提供第四层。

所以 (B) 在结构上是 (A) 的扩充：**任何 (B) 类机制内部都已经包含一条 (A) 类信号通路**（不然不知道何时触发），只是额外多了第四层——强制耦合。**这条额外的强制耦合，正是"机制 = 强制保证"这个核心性质的真正来源。**

两类都满足"运行在注意力预算之外、不可被提示词修改"，所以都属于机制层。但它们能解决的问题不同：

- (A) 解决的是"信号缺失"。它的失败模式是：当注意力本身已经塌掉时，再灌信号也接不住。
- (B) 解决的是"信号有了也接不住"。它在 (A) 之上加了第四层，把"做不做"这件事从模型那里收走，挪到外部。

这刀切清楚之后，主论文里那个例子才放得回正确的位置：失控重复属于 (B) 类，但非常容易被误当成 (A) 类去想（"是不是给模型一个'你在重复'的信号就行了？"）。一旦套错框架，论证就会卡住，方案也会跑偏——在失控重复的场景里，模型注意力已经塌掉，光给信号没用，必须有第四层的强制耦合。失控重复的完整论述见 [《主论文》](主论文.md) §3。

## 五、一般化：宏观上的注意力再分配

机制层下面要建的远不止失控重复这一个机制。一般而言，还有很多类似的小机制要做：什么叫逐段检测、什么叫时不时停下来看看自己有没有写重复、等等。

这些机制有一个共同特征：

> **用很多单个专门的动作，去替代一次性的注意力。**

一次性的注意力做不到的事情，拆成离散的、专门的检查点，反而能做到。

实际上这是**宏观上的注意力再分配**。注意：再分配的主体不是模型，是机制层的设计者——模型自己分配不好（这正是要解决的问题）；外部机制把"在什么节奏上、回到哪些检查点"这件事预先固化下来，让它在固定的节奏上反复回到该回到的地方。

这条原则适用于 (A) 与 (B) 两类：(A) 类信号机制可以按节奏定时给模型注入特定的检查点信号；(B) 类反射机制可以按节奏触发外部存活验证。比例随场景变化，但"碎片化的专门动作 替代 整片的注意力"这个底层模式是共通的。
